Transitions vs. Animations

Transitions vs. Animations


Transitions are triggered when there's a state change, e.g. a user hovers or clicks an element. Animations can be triggered at any point, without direct user input.

Transitions must be declared as defaults on the element in order to be triggered, while animations function independently of an element's default values.

Anatomy of a Keyframe

Animation @ block:

.animation {
  animation: [animation-name] [animation-duration] [animation-delay]
             [animation-timing-function] [iteration-count]
}

Only name and duration are required.

@keyframes [animation-name] {
  0% / from  { [properties-to-animate] }
  100% / to  { [properties-to-animate] }
} .spin-animation {
  background: $lookout-green;
  animation: spin 2s ease-in-out infinite;
} @keyframes spin {
  0%    { transform: rotate(0deg); }
  100%  { transform: rotate(360deg); }
}

Properties to know

animation-fill-mode: [backwards] [forwards] [both] [none]
animation-direction: [alternate] [reverse] [alternate-reverse]
steps(n) - granular, discrete control over keyframes. Great for animating image sprites.

Animation fill mode backwards, forwards and both will preserve different aspects of the animation keyframes

Animation direction reverse is used in many cases to simply and cheaply reverse a complex set of interactions
